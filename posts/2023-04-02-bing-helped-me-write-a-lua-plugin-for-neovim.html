Bing Helped Me Write a Lua Plugin for Neovim

Prompt:

Write me a neovim lua script that keeps a history of buffers for each window and has a command for going backwards and forwards in history.

Answer:

local history = {}
local current = 0

function AddToHistory()
  local bufnr = vim.fn.bufnr()
  local winid = vim.fn.win_getid()
  local entry = {bufnr, winid}
  table.insert(history, entry)
  current = #history
end

function GoBack()
  if current > 1 then
    current = current - 1
    local entry = history[current]
    vim.fn.bufload(entry[1])
    vim.fn.win_gotoid(entry[2])
  end
end

function GoForward()
  if current < #history then
    current = current + 1
    local entry = history[current]
    vim.fn.bufload(entry[1])
    vim.fn.win_gotoid(entry[2])
  end
end

vim.cmd([[autocmd BufEnter * lua AddToHistory()]])
vim.cmd([[command! -nargs=0 Back lua GoBack()]])
vim.cmd([[command! -nargs=0 Forward lua GoForward()]])

In less than 5 seconds Bing spits out the above. A quick read of it puts me in a state of mild shock. For someone who has never written Lua and is totally unfamiliar with the Neovim related APIs, this output is incredibly useful.

The Bad

I asked Bing if it was possible to do this in pure Lua:

Answer:

... You can also use vim.api.nvim_create_autocmd() to create autocommands

The answer includes a link to the Neovim Lua docs where I can learn more about this function.
